\subsection{Заполнение акта о выполнении работ}
Пользователь, ответственный за фиксацию фактически выполненных работах, может в системе отразить:
\begin{itemize}
\item  фактические сроки выполнения ремонтных работ, чтобы оценить выполнение плана;
\item  фактические трудозатраты на выполнение работ, чтобы получить расчет стоимости трудовых затрат на ремонты;
\item  фактические материальные затраты на выполнение работ, чтобы получить расчет стоимости материальных затрат на ремонты.
\end{itemize}

\subsubsection{Заполнение документа <<Акт выполнения работ>> подбором плановых ремонтов}

Пользователь, ответственный за отражение фактически выполненных работ, может заполнить данные в документе {\sl<<Акт выполненных работ>>} на основании ранне запланированных ремонтов, чтобы ускорить ввод данных и отразить факт исполнения запланированных ремонтов.

Например, в {\sl Августе 2016 г.} были запланированы ремонты по объекту ремонта {\sl <<Мини ТЭЦ «Океанариум»  Главный корпус>>} с инвентарным номером {\sl 000002026}. Пользоватею необходимо в докумене <<Акт выполнения работ>> заполнить таблицу выполненных ремонтов по всем запланированным ремонтам по этому объекту в отчете за {\sl Август 2016 г.}. Для этого пользователю необходимо воспользоваться кнопкой {\bf<<Подбор документов>>}.

Когда пользователь, ответственный за фиксацию фактически выполненных работ, выбирает в меню \textsl{Выполнение ТО и ремонтов $\Rightarrow$ Акт выполнения работ}, тогда откроется список документов {\sl<<Акт выполнения работ>>}. В форме списка пользователь нажимает кнопку {\bf<<Добавить>>}, тогда откроется форма нового документа. Пользователь заполняет поле {\sl<<Акт за период с>>}значением {\sl01.08.2016} поле {\sl по} значением {\sl 31.08.2016}, поле {\sl Период ремонтов} значением {\sl Август 2016 г.}
Когда ползователь нажимает на кнопку {\bf Подбор документов}, тогда откроется форма для подбора запланированных ремонтов для заполнения в документе. Система автоматически заполнит поле {\sl Сценарий планирования} значением текущего сценария планирования, а на закладке {\sl Настройки отбора} заполнит отбор по периоду планирования и ремонтному подразделению из шапки заполняемого документа.

Когда пользователь в поле {\sl Объект ремонта} вводи инвентарный номер {\sl 000002026} объекта ремонта, тогда в списке отбираются документы по этому объекту.

Например, в бухгалтерском учете за {\sl<<Август 2016 г>>} месяц были сформированы следующие обороты по складу {\sl <<Ясенчук С.В. (мастер цех)>>}:

\subsubsection{Сопоставление номенклатуры ТОиР с номенклатурой БУ} \label{sss:sopostavlenie_nomenklatury}

Пользователь, ответственный за формирование материального отчета, может заполнить в документе {\sl <<Отчет по материальным затратам>>} учетную номенклатуру, по которой ведется учет в системе бухгалтерского учета (<<Унифицированную номенклутуру>>), чтобы бухгалтер выполнил корректное списание номенклутары на основании данных материального отчета.

Например, в системе есть заполненный документ  {\sl<<Отчет по материальным затратам TO0000002 от 31.08.2016  0:00:00>>} с заполненной табличной частью {\sl <<Расход материалов>>} см. раздел \ref{sss:zapolnenie_po_fakt_vyp_rabotam}. В отчете используется номенклатура, у которой заполнен реквизит {\sl<<Унифицированная номенклатура>>}.

\begin{figure}[H]
\hspace*{-1.5cm}{\includegraphics[width=19cm]{pic/zapolnenie_univers_nom}}
\caption{Пример заполнения реквизита <<Универсальная номенклатура>>}
\label{ris:zapolnenie_univers_nom}
\end{figure}

Поьзователь открывает документ  {\sl<<Отчет по материальным затратам TO0000002 от 31.08.2016  0:00:00>>} и в форме документа нажимает на кнопку {\bf <<Заполнить соответствие номенклатуры БУ>>}, тогда система автоматически заполнит соответствующую учетную номенклатуру из реквизита {\sl<<Универсальная номенклатура>>} элемента, указанного в колонке {\sl<<Номенкалтура>>}.

\subsection{План/фактный анализ выполнения программы ремонтов}

\subsubsection{Анализ соответствия фактических сроков выполнения ремонтов запланированным}

Пользователь, ответственный за своевременность исполнения плана ремонтов, может в системе сфомировать отчет для анализа своевременности выполнения ремонтов.

Например, в системе был спланирован в рамках сценария { \sl ТОиР корр.№1 БП 2016} {\sl Ремонт эл. двигателя сетевых насосов "Центральная -ДВФУ" СН-4} {\sl << Мини ТЭЦ "Центральная" Насос сетевой системы ОВ,  N-250 кВт, TP 300-670/4A-F-A,  DBUA  № 4 [000001357]>>} в {\sl Августе 2016 г.}. Факт выполнения ремонта можно посмотреть в рабочем месте {\slПланирование} вариант {\sl График работ}.

Пользователь переходит в меню \textsl{Планирование ремонтов $\Rightarrow$ Плановые ремонты (Планирование)}, тогда открывается форма обработки <<Планирование>>. Пользователь заполняет на форме поле {\sl Вариант} значением {\sl График работы},  поле {\sl Сценарий (список)} значением {\sl ТОиР корр.№1 БП 2016},  поле {\sl Планируемый период} значением {\sl Август 2016 г.},  поле {\sl Периодичность} значением {\sl День} и нажимает на кнопку {\bf Сформировать}. Тогда будет сформирован отчет с данными о сроках запланированных и фактически выполненных ремонтах.

Например, в системе был спланирован в рамках сценария { \sl ТОиР корр.№1 БП 2016} {\sl Ремонт эл. двигателя сетевых насосов "Центральная -ДВФУ" СН-4} {\sl << Мини ТЭЦ "Центральная" Насос сетевой системы ОВ,  N-250 кВт, TP 300-670/4A-F-A,  DBUA  № 4 [000001357]>>} в {\sl Августе 2016 г.} на общую сумму {\sl 549 329 руб} . Фактические прямые затраты на этот ремонт можно увидеть в рабочем месте {\slПланирование} вариант {\sl Затраты}.

Пользователь заполняет на форме поле {\sl Вариант} значением {\sl Затраты},  поле {\sl Сценарий} значением {\sl ТОиР корр.№1 БП 2016},  поле {\sl Планируемый период} значением {\sl Август 2016 г.},  поле {\sl Периодичность} значением {\sl Месяц} и нажимает на кнопку {\bf Сформировать}. Тогда будет сформирован отчет с данными о плновой сумме и фактической стоимости выполненных ремонтов.
